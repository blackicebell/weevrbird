# Weevrbird Terms Of Use Draft

Date: 2026-07-19

This is draft copy for store preparation. It is not legal advice. Review and update it before public launch, especially before enabling production accounts, public contributions, moderation, backend sync, or paid features.

## Overview

Weevrbird helps people read finite daily editions, follow Smartfeeds, save useful pieces, and contribute notes, questions, recommendations, links, and long reads.

By using Weevrbird, users should agree to use the app in a way that keeps contributions useful, lawful, and respectful.

## Accounts And Identity

The current build uses an email-link identity flow with an optional pen name.

Users are responsible for using an email address they control and for keeping their device secure.

Pen names should not impersonate another person, organization, or source.

## Contributions

Users may create contributions such as notes, questions, recommendations, links, and long reads.

Contributions should:

- Add useful context
- Avoid spam, harassment, threats, hate, sexual exploitation, illegal instructions, or intentionally misleading claims
- Respect other people's privacy
- Avoid posting private information without permission
- Include source context when sharing links, articles, images, videos, or resources

Contributions are saved privately first in the current app flow. A user chooses where a contribution belongs before it becomes visible in a Smartfeed.

## External Sources And Links

Weevrbird may show attributed external source cards and links.

External links may open outside Weevrbird. Weevrbird does not control external websites, their content, or their privacy practices.

Users should not assume that Weevrbird endorses every claim made by an external source.

## Moderation

Weevrbird should reserve the right to remove, limit, or refuse content that violates these terms or harms the usefulness and safety of the app.

Production moderation, reporting, blocking, and appeal processes still need implementation before public launch.

## User Content Rights

Users should keep ownership of the content they create.

By choosing to make a contribution visible in Weevrbird, users should give Weevrbird permission to display, organize, and distribute that contribution inside the app.

Users should only contribute content they have the right to share.

## Local Data And Reset

The current build stores most app state locally on the device.

Reset Weevrbird clears local setup, saved items, reading history, and contribution drafts on that device.

Production account deletion and server-side data deletion need to be implemented or clearly handled through support before public launch.

## Disclaimers

Weevrbird is an information and discovery product. It should not be positioned as a source of professional legal, medical, financial, safety, or emergency advice.

Users should verify important information with original sources before acting on it.

## Changes

These terms may need to change as Weevrbird adds backend accounts, live source ingestion, moderation, notifications, analytics, paid features, or other production systems.

## Contact

Support: support@weevrbird.app
